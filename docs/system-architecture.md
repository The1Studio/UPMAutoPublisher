## Architecture Overview

### Design Philosophy

UPM Auto Publisher follows a **dispatcher-handler architecture** where:
- **Lightweight dispatchers** in target repositories detect changes and trigger events
- **Centralized handler** in UPMAutoPublisher executes all publishing logic
- **Single source of truth** for business logic (handler)
- **Stable interface** between dispatcher and handler (rarely changes)

**Key Benefits**:
- âœ… Update logic once (central handler) instead of 27 times (each repo)
- âœ… Consistent behavior across all repositories
- âœ… Easy testing (test handler without affecting repos)
- âœ… Gradual rollout capability
- âœ… Simple rollback (revert handler changes)

### Security Layers

1. **Input Validation Layer**
   - Semver regex validation
   - Package name validation
   - URL validation
   - Markdown injection prevention

2. **Command Injection Prevention Layer**
   - jq-only JSON construction
   - All variables quoted
   - No string interpolation
   - Parameterized commands

3. **Authentication Layer**
   - GitHub Actions OIDC
   - NPM token authentication
   - GH_PAT validation
   - Secure token handling

4. **Rate Limiting Layer**
   - npm API rate limiting
   - Exponential backoff
   - Discord webhook rate limits
   - Gemini API rate limits

5. **Audit Layer**
   - Comprehensive logging
   - 90-day retention
   - No sensitive data in logs
   - GitHub Actions artifacts

6. **Network Security Layer**
   - HTTPS only (registry, API calls)
   - Registry health checks
   - Timeout enforcement
   - Error handling

**Vertical Scaling** (Increase runner resources):
